💡 Parte Teórica

📚 ¿Qué es?

"El manual de instrucciones de tu software" - Como el manual de un electrodoméstico pero para programas.

❓ ¿Por qué es importante?

  • Para que OTROS configuren sin ti
  • Para que TÚ recuerdes cómo lo hiciste
  • Para cuando algo falla

📏 Estándar IEEE 829

Versión simplificada:

  1. Portada
  2. Introducción
  3. Requisitos
  4. Configuración
  5. Verificación
  6. Apéndice

✨ Ejemplo: Cómo escribir un buen paso

❌ MAL

"Configurar base de datos."

✅ BIEN

4.3 Configurar conexión a base de datos

a) Abrir archivo .env (carpeta raíz)
b) Buscar línea: DB_CONNECTION=
c) Escribir después del = la URL
d) Guardar (Ctrl+S)
✅ Resultado: App se conecta sin errores

✅ Reglas de Oro y Checklist

🎯 REGLA DE ORO: Documenta como si la persona que lo lea no supiera nada y estuviera enojada.

📋 Checklist de calidad

  • ¿Tiene portada con tus datos?
  • ¿Tiene fecha y versión?
  • ¿Los pasos son fáciles de seguir?
  • ¿Alguien sin conocimientos podría seguirlos?
  • ¿Incluye requisitos de hardware/software?
  • ¿Tiene pruebas para verificar que funciona?
  • ¿Incluye solución a problemas comunes?
  • ¿Tiene información de contacto?
  • ¿Usa formato claro?
    • ⚠️ ERRORES COMUNES:
    • Suponer que el lector sabe cosas
    • Instrucciones vagas ("configurar el sistema")
    • No incluir qué hacer si falla
    • Fechas desactualizadas
